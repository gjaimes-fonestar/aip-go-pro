# AIP Go Pro

Desktop application for professional management and control of AIP (Audio Interface Protocol) devices. Built with Electron, React, and TypeScript.

---

## Requirements

### Runtime

| Platform | Architecture | Format |
|----------|-------------|--------|
| Windows 10/11 | x64 | NSIS installer |
| Linux | x64 | AppImage / deb |

### Development

| Tool | Minimum version | Purpose |
|------|----------------|---------|
| Node.js | 18 LTS | JavaScript runtime |
| npm | 9+ | Package manager |
| CMake | 3.20+ | Build the AIP daemon |
| Git | any | Version control |

> The `aip-daemon` binary lives in the sibling project `../aip-go-services/`. It must be compiled and copied into `resources/bin/` before packaging the Electron app.

## Project Structure

```
aip-go-pro/
├── src/
│   ├── main/           # Electron main process — daemon lifecycle, WebSocket server, IPC
│   ├── preload/        # Electron preload — secure IPC bridge to the renderer
│   ├── renderer/       # React UI — pages, components, stores, router
│   └── shared/         # Shared TypeScript types (IPC contracts)
├── resources/
│   ├── bin/
│   │   ├── linux/      # aip-daemon binary for Linux
│   │   └── windows/    # aip-daemon.exe binary for Windows
│   └── icons/          # Application icons
├── out/                # Compiled output (generated by build)
├── dist/               # Packaged installers (generated by package)
├── electron.vite.config.ts
├── electron-builder.yml
└── package.json
```

## Setup

### 1. Clone and install dependencies

```bash
git clone <repo-url> aip-go-pro
cd aip-go-pro
npm install
```

### 2. Build and install the daemon (Linux)

The daemon must be built from the sibling project before running or packaging.

```bash
# Build the daemon (requires CMake + Go in ../aip-go-services)
npm run daemon:build

# Copy the compiled binary into resources/bin/linux/
npm run daemon:copy:linux
```

Both steps combined:

```bash
npm run daemon:install:linux
```

### 3. Build and install the daemon (Windows)

```bash
npm run daemon:build
npm run daemon:copy:win
```

## Running in Development

```bash
npm run dev
```

This starts the Electron app with Vite hot-reload. The renderer dev server listens on all interfaces (`0.0.0.0`) so the UI is also reachable from other devices on the same LAN, which is useful for testing on hardware.

---

## Building for Production

Compile TypeScript and bundle all processes:

```bash
npm run build
```

Output is written to `out/`.

Preview the production build without packaging:

```bash
npm run preview
```

## Packaging

Produce distributable installers in `dist/`.

| Command | Output |
|---------|--------|
| `npm run package:linux` | AppImage + `.deb` (x64) |
| `npm run package:win` | NSIS installer (x64) |
| `npm run package` | Both platforms |

The daemon binary must be present in `resources/bin/` before running any package command.

## Architecture Overview

```
Renderer (React)
    │  IPC (contextBridge)
    ▼
Preload script
    │  IPC (ipcMain / ipcRenderer)
    ▼
Main process (Electron)
    │  WebSocket
    ▼
aip-daemon (Go binary)
```

- **Main process** spawns the `aip-daemon` binary and exposes it via a local WebSocket server.
- **Preload** provides a safe IPC bridge so the renderer never gets direct access to Node.js APIs.
- **Renderer** communicates with the daemon through the IPC bridge (Electron) or directly via WebSocket when running in a browser (using `browserPolyfill.ts`).
